# Harness Validation Report: Metric Sanity Checks

## Overview
This report documents the validation of the core benchmarking metrics used in the Superconducted Noise Engine. As part of the harness validation process, we implemented a suite of sanity checks to ensure that the mathematical properties of our metrics—Hellinger Distance, Kullback-Leibler (KL) Divergence, State Fidelity, and R2 Score—behave exactly as expected under controlled boundary conditions.

## Validated Properties
We introduced 10 distinct tests in `tests/test_harness_validation.py` to cover the following mathematical properties, using mocked `SimulationResult` objects:

### 1. Identity Tests
- **Hellinger Distance & KL Divergence:** Verified that comparing identical probability distributions yields exactly `0.0`.
- **State Fidelity:** Confirmed that the fidelity of a quantum density matrix with itself evaluates to `1.0`.

### 2. Symmetry & Asymmetry
- **Symmetry:** Hellinger Distance and State Fidelity were confirmed to be perfectly symmetric (i.e., evaluating `A vs B` yields the same result as `B vs A`).
- **Asymmetry:** KL Divergence was correctly validated as an asymmetric metric. Swapping the engine and reference distributions produced the expected distinct values.

### 3. Bounds and Reference Values
- **Disjoint Bounds:** Hellinger Distance correctly bounded at its maximum value of `1.0` when evaluating completely disjoint distributions (e.g., pure `00` vs pure `11`).
- **Orthogonality:** State Fidelity correctly returned `0.0` for completely orthogonal quantum states.
- **R2 Score Logic:** The R2 Score implementation was validated against its reference behavior. It correctly returns `1.0` for perfectly matching distributions. Crucially, we verified its fallback logic: it gracefully returns `0.0` when the reference distribution is uniform (constant) but the engine output deviates entirely.

## Conclusion
All 10 validation tests passed successfully against the implementations in `src/superconducted/benchmarks/metrics.py`. The metrics are robust, properly object-oriented, and adhere strictly to their theoretical mathematical constraints. This ensures that downstream benchmarking results generated by the engine will be reliable and mathematically sound. No anomalies or edge-case failures were detected during this validation phase.
